# Contributing

Gracias por tu interés en mejorar Control Horario Digital.

## Cómo contribuir

- Reportar errores
- Proponer mejoras
- Enviar correcciones
- Mejorar documentación

## Antes de contribuir

- Revisa la documentación disponible
- Comprueba si el problema o mejora ya ha sido detectado
- Mantén los cambios enfocados y bien delimitados

## Recomendaciones

- Respeta la estructura y estilo del proyecto
- Prioriza robustez, seguridad y claridad
- Evita cambios innecesarios fuera del alcance de la mejora
- Si el cambio afecta a comportamiento sensible, documenta el motivo

## Reporte de bugs

Incluye siempre:

- Descripción del problema
- Pasos para reproducirlo
- Resultado esperado
- Resultado actual
- Entorno de ejecución

## Pull requests

Si envías una contribución:

- Explica qué problema resuelve
- Mantén el cambio lo más pequeño posible
- Evita mezclar refactor y funcionalidad nueva en la misma propuesta

## Seguridad

Si detectas una vulnerabilidad, sigue la política indicada en `SECURITY.md` y evita publicarla de forma abierta antes de comunicarla.
